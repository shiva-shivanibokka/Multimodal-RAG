# Audit report

SAFE-tier bug audit of the Multimodal RAG Trust Layer (FastAPI backend on Cloud Run + Next.js frontend). Findings grouped by severity, each with file:line, production impact, and status. `FIXED-safe` = applied in this pass (low-risk, no behavior change beyond the fix). `IN-PLAN` = tracked in `PLAN.md`, not applied here.

## HIGH

- **Model singleton double-init race** — `backend/app/index/embedders.py:36-47`, `backend/app/index/rerank.py:18-22`, `backend/app/verify/nli.py:31-44`, `backend/app/ingest/ocr.py:28-34`, `backend/app/ingest/tables.py:24-28`. Each is a bare `if _model is None: _model = Load(...)` with no lock. Cloud Run's `concurrency=4` can dispatch 4 requests into a cold container at once; all 4 see `None` and load their own copy of a multi-hundred-MB model concurrently, multiplying peak RSS on an 8Gi instance. IN-PLAN.
- **`get_index` concurrent build race** — `backend/app/session.py:35-49`. `if "index" not in session:` is check-then-act with no lock; two `/answer` calls hitting the same brand-new session in parallel both build the index (OCR every page + 3 FAISS builds) simultaneously, doubling CPU/memory for that session and racing to overwrite `session["index"]`. IN-PLAN.
- **`/ingest` page-count amplification (DoS)** — `backend/app/main.py:59-64` loop, fed by `backend/app/ingest/loader.py:38-47`. The only cap is 25MB of bytes (`main.py:41-50`); a small text-only PDF can legitimately contain thousands of pages. Each page runs OCR (if scanned) and `extract_tables` (which OCRs again, see below) — a few-hundred-KB file can trigger hours of CPU. IN-PLAN.
- **Redundant OCR (compounds the DoS above)** — `backend/app/index/store.py:87-88` re-OCRs every page (`ocr_page`) to build the caption index, even though `/ingest` already OCR'd scanned pages into `page["text_blocks"]` at `backend/app/main.py:60-61`. Separately, `backend/app/main.py:62` calls `extract_tables(page["image_png"])` on **every** page unconditionally, and `extract_tables` (`backend/app/ingest/tables.py:65-74`) runs its own docTR OCR pass via `img2table`'s `DocTR` backend regardless of whether the page has text. Net effect: a scanned page can be OCR'd 2-3 times per ingest. IN-PLAN.

## MEDIUM

- **`/answer` has no request-size cap** — `backend/app/schemas.py` (`AnswerRequest`). `question`, `model`, `api_key` are unbounded `str` — a multi-MB `question` is accepted, embedded, sent to the NLI cross-encoder, and forwarded to the LLM provider at the caller's token cost. IN-PLAN.
- **Malformed provider response → 500** — `backend/app/generate/providers.py:58` (`r.json()["choices"][0]["message"]["content"]`) and `:65` (`r.json()["content"]`). A 200 response with an unexpected shape (e.g. provider returns `{"choices": []}` or a moderation-blocked empty payload) raises an uncaught `KeyError`/`IndexError`, surfacing as an unhandled 500 instead of the existing `ProviderError` mapping used everywhere else in this module. IN-PLAN.
- **Hybrid grounding gate ignores lexical evidence** — `backend/app/generate/answer.py:84-91` (`_grounding_top`), used at `:104-106`. In `hybrid` mode the refusal gate only checks `index.dense(...)` (bge cosine); a query that's a strong exact/keyword match (high BM25, low dense cosine) can be wrongly refused even though `retrieve()` at `:108` would have found it via RRF fusion. IN-PLAN.
- **`_dedupe_columns` can re-collide** — `backend/app/ingest/tables.py:31-46`. Suffixing only checks `seen` (the *original* header names), not the *already-emitted* output set: `["A", "A", "A.1"]` → first "A" kept, second "A" becomes "A.1", third "A.1" is untouched (not in `seen`) → duplicate `"A.1"` column, which breaks `pd.to_numeric(df[col], ...)` (`tables.py:59`, ambiguous `df[col]` on a dup-named column raises) downstream in `_rows_to_dataframe`. IN-PLAN.
- **Dropdown not keyboard-operable** — `frontend/components/Dropdown.tsx:51-68`. Options (`<li role="option">`) only bind `onClick`; no arrow-key movement, no Enter/Space-to-select, no roving `tabIndex`. A keyboard-only user can open the menu (`Enter`/`Space` on the trigger button) but cannot choose an option — an a11y regression vs. the native `<select>` it replaced. IN-PLAN.
- **Proxy routes have no fetch timeout** — `frontend/app/api/answer/route.ts:4-8`, `frontend/app/api/eval-report/route.ts:3-5`, `frontend/app/api/ingest/route.ts:4-8`, `frontend/app/api/page/[session]/[page]/route.ts:7-9`. None pass an `AbortSignal`; a hung backend (e.g. mid-NLI-verification on a cold container) leaves the Next.js function running until the platform's hard timeout instead of failing fast with a 504. IN-PLAN.

## LOW

- **Backend deps mostly unpinned** — `backend/requirements.txt:2-27`. Only `torch`/`torchvision` are pinned; `fastapi`, `sentence-transformers`, `img2table`, `faiss-cpu`, etc. float, so a fresh build can silently pull a breaking transitive update. IN-PLAN.
- **sessionStorage BYOK key is write-only** — `frontend/app/page.tsx:104-108` (`handleApiKeyChange`). Writes to `sessionStorage` on every keystroke but nothing ever reads it back (no lazy-init of `apiKey` state on mount), so it doesn't actually survive a refresh — the write exists but delivers no persistence, just needless storage churn. IN-PLAN.
- **`/ingest` echoes the raw parse exception** — `backend/app/main.py:51-54` (`detail=f"could not parse document: {exc}"`). Returns the raw `Exception` string (potentially internal library/path detail) straight to the client instead of logging server-side and returning a generic message. IN-PLAN.
- **Stale Anthropic model list** — `frontend/app/page.tsx:37-41` (`MODELS.anthropic`). Stops at the Claude 3.x family; no current-generation entries. IN-PLAN.
- **No test for `lib/bbox.ts`** — `frontend/package.json:10` runs `node --test lib/**/*.test.mts`, but no `lib/bbox.test.mts` exists, so `bboxToOverlayRect` (including its `natural.width/height === 0` divide-by-zero guard at `frontend/lib/bbox.ts:12-13`) has zero test coverage. IN-PLAN.

## Auto-fixed in this pass (SAFE tier)

1. **Constant-time token compare** — `backend/app/main.py:15-17` (`require_token`). `authorization != f"Bearer {...}"` was a variable-time `!=`; a network-position attacker could in principle use timing to help brute-force `BACKEND_TOKEN`. Now uses `hmac.compare_digest`. FIXED-safe.
2. **Provider validated at the schema boundary** — `backend/app/schemas.py` (`AnswerRequest.provider`). Was a bare `str`; `route_provider()` (`backend/app/generate/providers.py:20-25`) raised a plain `ValueError` for unknown providers, which FastAPI turns into an unhandled 500. Now `Literal["openai", "groq", "gemini", "anthropic"]`, so an invalid provider is rejected with a clean 422 before it reaches `answer_question`. Verified no test or app code sets `provider` to a non-listed value (`test_answer.py`, `test_health.py` both use `"groq"`; `test_providers.py`'s `test_unknown_provider_raises` calls `route_provider()` directly, not through the API, so it's unaffected). FIXED-safe.
3. **Offline model loading on Cloud Run** — `backend/Dockerfile` (`ENV` block). Added `HF_HUB_OFFLINE=1` / `TRANSFORMERS_OFFLINE=1`. Model weights are baked into the image at build time (see the `RUN python -c "..."` warm-load block); without these, a cold container still does a network metadata round-trip per model to check for updates before falling back to cache, adding latency and a hard external dependency to every cold start. FIXED-safe.
4. **Citation image error state** — `frontend/components/CitationViewer.tsx`. The `<img>` had no `onError`; a failed page-image fetch (e.g. proxy 502, expired session) left the "Loading page image…" spinner running forever with no user feedback. Added an `error` state, `onError` handler, and a fallback message in place of the spinner; reset alongside `natural`/`rendered` in the existing `[citation.page]` effect. Bbox overlay logic untouched. FIXED-safe.
5. **Cancel unconsumed proxy body** — `frontend/app/api/page/[session]/[page]/route.ts:10-13`. The `!r.ok` branch discarded `r` without draining `r.body`, leaking the backend connection/stream on every non-2xx page-image fetch. Now calls `r.body?.cancel()` before returning. FIXED-safe.
6. **Dropped a deprecating model** — `frontend/app/page.tsx` (`MODELS.gemini`). Removed `gemini-1.5-flash` (on Google's deprecation track); `gemini-2.0-flash`, `gemini-2.5-flash`, `gemini-2.5-pro` remain. FIXED-safe.

Verification: backend fast suite (`test_health.py`, `test_providers.py`, `test_table_answer.py`) — 19 passed. `npm run build` — compiled and typechecked clean, all 4 API routes + `/` + `/eval` built.

## Verified clean / no issue found

- **BYOK key never leaked server-side.** `backend/app/generate/providers.py:10-12,29-30` — `ProviderError`'s docstring and `_post`'s comment both call out that `api_key`/headers are never included in error messages; confirmed no log/print of `api_key` or `headers` anywhere in `providers.py`, `answer.py`, or `main.py`. The key travels browser → `/api/answer` (Next proxy, not logged) → backend `/answer` body → provider `Authorization`/`x-api-key` header only.
- **No path traversal.** `backend/app/main.py:81-93` (`get_page`) and `backend/app/session.py` — `session_id` is a `uuid4().hex` dict key (`session.py:22`) and `page_index` is bounds-checked (`0 <= page_index < len(pages)`) against an in-memory list; neither ever touches a filesystem path built from user input.
- **No SSRF.** Every outbound URL is either a hardcoded constant (`_OPENAI_COMPAT` dict + the anthropic URL in `providers.py:3-7,24`, selected only via the now-`Literal`-validated `provider` field) or a server-controlled env var (`process.env.BACKEND_URL` in the 4 Next.js proxy routes) — no user input ever reaches a fetched URL.
- **LRU session eviction is correct.** `backend/app/session.py:14,20-32` — `OrderedDict` with `move_to_end` on every `get_session`/`get_index` hit and `popitem(last=False)` (oldest) eviction on overflow at `create_session`; standard, correct LRU.
- **Citation/Claim/AnswerResponse types match frontend↔backend.** `backend/app/schemas.py` vs `frontend/lib/backend.ts:1-4` — field names and shapes (`page:number`/`int`, `bbox:number[]`/`list[float]`, `snippet`, `claims`, `citations`, `refused`) line up exactly.
- **Next.js 16 async `params` handled correctly.** `frontend/app/api/page/[session]/[page]/route.ts:2-5` — `params` is typed `Promise<{...}>` and `await`ed before use, matching the Next 16 App Router convention (confirmed against `node_modules/next/dist/docs/` per `frontend/AGENTS.md`).
- **NLI label order isn't hardcoded blindly.** `backend/app/verify/nli.py:31-44` — `_get_model` reads `model.config.id2label` at load time and asserts exactly one `"entailment"` label instead of assuming index 1, so a model swap that reorders labels fails loudly instead of silently flipping supported/unsupported.
- **Upload size cap defends against a lying/missing `Content-Length`.** `backend/app/main.py:41-50` — checks the header when present, then re-checks actual bytes read (capped read of `cap + 1`) regardless, so a request that omits or lies about `Content-Length` still can't exceed the 25MB cap.
